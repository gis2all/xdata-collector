# 自动任务 `group_name` 设计说明

## 概述

为自动任务新增一个单值字段 `group_name`，用于给任务做更稳定的单分类分组。它和现有 `tags` 并列存在，但语义不同：

- `group_name`：单个分组值，用于任务归类
- `tags`：多个标签值，用于更细粒度标记

这次改动只面向自动任务 `job` 本身，不把分组放进任务包，也不替代现有 tags。

## 目标

- 在自动任务右侧编辑区支持查看和编辑 `group_name`
- 在自动任务表格中新增“分组”列
- 让 `group_name` 通过前后端接口和工作区存储完整流转
- 保持现有任务包、tags、运行逻辑不变

## 核心决策

### 1. 字段归属

`group_name` 归属于自动任务 `job`，不是任务包字段。

原因：

- 同一个任务包可能被多个任务复用，但不同任务可能需要不同分组
- 分组更接近调度和管理视角，不是搜索规格本身
- 这样可以避免“改一个任务包，连带改掉其他任务分组”的副作用

### 2. 字段形态

代码字段名使用英文 `group_name`。

数据类型为单值字符串，可为空：

- 前端编辑态可以是空字符串
- 存储和接口层统一归一化为 `string | null`
- 空值展示统一为 `--`

### 3. 与 tags 的关系

`group_name` 和 `tags` 同时保留，互不推导：

- 不从 tags 自动生成 group
- 不把 group 拆成单元素 tags
- 不把 tags 限制为单值

## 数据模型

### 前端

`web-ui/src/api.ts`

- `JobRecord` 新增 `group_name?: string | null`
- `createJob(...)` payload 新增 `group_name?: string | null`
- `updateJob(...)` payload 新增 `group_name?: string | null`

### 后端接口

自动任务相关接口透传 `group_name`：

- `POST /jobs/create`
- `POST /jobs/{id}/update`
- `GET /jobs`
- `GET /jobs/{id}`

### 工作区存储

`group_name` 保存在 workspace 的 `jobs[]` 注册信息中，而不是任务包 JSON 中。

这意味着：

- 创建任务时写入 job registry
- 更新任务时只更新对应 job
- 老数据没有该字段时按 `null` 处理
- 任务包导入、保存、覆盖时不读写该字段

## 交互与展示

### 右侧任务编辑区

在自动任务右侧“当前任务”基础信息区新增 `分组` 输入框。

推荐位置：

1. `任务名称`
2. `分组`
3. `执行间隔（分钟）`
4. `启用`

行为：

- 输入框允许留空
- 保存时做 `trim`
- 为空时按 `null` 持久化
- 加载已有任务时回填当前 `group_name`
- 新建任务默认空值

### 自动任务表格

新增“分组”列，放在：

`任务 -> 任务包 -> 分组 -> 任务标签 -> 间隔 -> 状态 -> 下次运行 -> 最近运行`

展示规则：

- 有值时显示 `group_name`
- 无值时显示 `--`
- 不做彩色 pill，保持单值文本展示，和 tags 视觉区分

## 搜索与筛选

自动任务列表现有关键字搜索应纳入 `group_name`，这样用户按分组名也能搜到对应任务。

这次不新增单独的“按分组筛选器”，只复用现有任务搜索框。

## 与任务包的边界

因为 `group_name` 是 job 级字段，所以任务包相关操作保持下面规则：

- 从任务包加载草稿：不覆盖当前任务的 `group_name`
- 保存到任务包：不把 `group_name` 写入 pack
- 另存为新任务包：不把 `group_name` 写入 pack
- 从文件导入任务包：只影响 pack 内容，不影响 `group_name`

这样可以保证“搜索内容模板”和“任务管理分组”两套概念不串线。

## 兼容性

- 历史 jobs 没有 `group_name` 时，读取结果应为 `null`
- 老工作区文件不需要迁移脚本，走读取时兜底即可
- 不改 raw / curated 结果表，不把 `group_name` 继承到采集结果

## 非目标

这次不做下面这些内容：

- 不把分组做成多值数组
- 不把分组做成任务包字段
- 不新增分组颜色体系
- 不新增按分组聚合、折叠、批量操作
- 不修改 tags 的解析、保存、展示语义

## 测试范围

### 后端

- 创建 job 时可保存 `group_name`
- 更新 job 时可修改和清空 `group_name`
- `list_jobs` / `get_job` 返回 `group_name`
- 历史缺失字段时返回 `null`
- 列表搜索可匹配 `group_name`

### 前端

- Jobs 页右侧表单可编辑 `group_name`
- 新建、保存、重新加载后 `group_name` 正常回填
- Jobs 表格出现“分组”列，位置正确
- 空分组显示 `--`
- 搜索框可命中 `group_name`

## 验收口径

用户可以为每个自动任务单独设置一个分组名，并且：

- 在任务表里直接看到它
- 在右侧编辑区修改后立即可保存和回显
- 不会污染任务包
- 不会和 tags 混成同一种东西
